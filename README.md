# Airport Database System

## Project Overview
This project is a relational database system designed to support airport booking, flight scheduling, passenger management, payment tracking, frequent flyer records, route management, and operational reporting.

## Database Structure
The database includes 8 connected tables: Airplane, Airport, Flight, Booking, Passenger, Frequent Flyer, Payment, and Route. The schema uses primary keys, foreign keys, identity columns, and table relationships to maintain data integrity and model real-world airport operations.

## Key Features
- Designed a relational database schema for airport operations
- Created tables for flights, passengers, bookings, airports, aircraft, payments, routes, and frequent flyer records
- Implemented primary and foreign key relationships across the database
- Added sample data to simulate booking, payment, and scheduling workflows
- Built SQL queries for reporting and business analysis
